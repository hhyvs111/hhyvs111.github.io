基本UDP套接字编程

《Unix网络编程：卷1》``udp``posts

udp的特性就不再介绍。

1 概述

主要流程：

2 recvfrom和sendto函数

UDP不同于TCP，客户不需要与服务器建立连接，只需要指定目的地址就可以发送数据。发送数据使用sendto，接收使用recvfrom。这两个函数类似于标注的read和write。

2.1 函数原型

#include <sys/socket.h>
ssize_t recvfrom(int sockfd, void *buf, size_t len, int flags,
                        struct sockaddr *src_addr, socklen_t *addrlen);
ssize_t sendto(int sockfd, const void *buf, size_t len, int flags,
                      const struct sockaddr *dest_addr, socklen_t addrlen);

2.2 参数详解

前三个参数就不再介绍了，等同与read和write函数的三个参数。

1）flags

一般置为0.

2）src_addr和 addrlen

  • recvfrom的最后两个参数类似与accept的最后两个参数：返回其中套接字地址结构的内容告诉我们是谁发送了数据报（UDP）或是谁发起了连接（TCP）。
  • sendto的最后两个参数类似与connect的最后两个参数：调用时其中套接字地址结构被我们跳入数据报将发往（UDP）或与之建立连接（TCP）的协议地址。

3 例子

服务端：

#include    "unp.h"

int
main(int argc, char **argv)
{
    int                 sockfd;
    struct sockaddr_in  servaddr, cliaddr;

    sockfd = Socket(AF_INET, SOCK_DGRAM, 0); //注意这里是SOCK_DGRAM,而TCP是SOCK_STREAM。

    bzero(&servaddr, sizeof(servaddr));
    servaddr.sin_family      = AF_INET;
    servaddr.sin_addr.s_addr = htonl(INADDR_ANY);
    servaddr.sin_port        = htons(SERV_PORT);

    Bind(sockfd, (SA *) &servaddr, sizeof(servaddr));

    dg_echo(sockfd, (SA *) &cliaddr, sizeof(cliaddr));//函数实现可看书
}

客户端：

#include    "unp.h"

int
main(int argc, char **argv)
{
    int                 sockfd;
    struct sockaddr_in  servaddr;

    if (argc != 2)
        err_quit("usage: udpcli <IPaddress>");

    bzero(&servaddr, sizeof(servaddr));
    servaddr.sin_family = AF_INET;
    servaddr.sin_port = htons(SERV_PORT);
    Inet_pton(AF_INET, argv[1], &servaddr.sin_addr);

    sockfd = Socket(AF_INET, SOCK_DGRAM, 0);

    dg_cli(stdin, sockfd, (SA *) &servaddr, sizeof(servaddr));

    exit(0);
}

套接字选项

《Unix网络编程：卷1》``posts``SOCKET

这一章主要介绍一下几个函数。

1 getsockopt函数和setsockopt函数

  • getsockopt()函数用于获取任意类型、任意状态套接口的选项当前值，并把结果存入optval。在不同协议层上存在选项，但往往是在最高的“套接口”层次上，设置选项影响套接口的操作，诸如操作的阻塞与否、包的选径方式、带外数据的传送等。

  • setsockopt()函数用于任意类型、任意状态套接口的设置选项值。尽管在不同协议层上存在选项，但本函数仅定义了最高的“套接口”层次上的选项。选项影响套接口的操作，诸如加急数据是否在普通数据流中接收，广播数据是否可以从套接口发送等等。

1.1 函数原型

#include <sys/types.h>          /* See NOTES */
#include <sys/socket.h>

int getsockopt(int sockfd, int level, int optname,void *optval, socklen_t *optlen);
int setsockopt(int sockfd, int level, int optname, const void *optval, socklen_t optlen);

1.2 参数详解

  • sockfd：一个标识套接口的描述字。
  • level：选项定义的层次。支持的层次仅有SOL_SOCKET和IPPROTO_TCP。
  • optname：需获取的套接口选项。
  • optval：指针，对于getsockopt()，指向返回选项值的缓冲。对于setsockopt()，指向包含新选项值的缓冲。
  • optlen：指针，对于getsockopt()，作为入口参数时，选项值的最大长度。作为出口参数时，选项值的实际长度。对于setsockopt()，现选项的长度。

2 fcntl函数

可执行各种描述符控制操作，比如设置非阻塞式I/O。

2.1 函数原型

#include <unistd.h>
#include <fcntl.h>
int fcntl(int fd, int cmd); 
int fcntl(int fd, int cmd, long arg); 
int fcntl(int fd, int cmd, struct flock *lock);

返回：若成功则取决于cmd，若出错则为-1。 下列三个命令有特定返回值：

  • F_DUPFD 返回新的文件描述符
  • F_GETFD 返回相应标志
  • F_GETFL , F_GETOWN 返回一个正的进程ID或负的进程组ID

2.2 参数详解

主要看一下cmd的值：

  • F_SETFL 设置文件状态标记
  • F_GETFL 获得文件状态标记
  • F_DUPFD 复制一个现有的描述符
  • F_GETFD 获得文件描述符标记
  • F_SETFD 设置文件描述符标记
  • F_GETOWN 获得异步I/O所有权
  • F_SETOWN 设置I/O所有权

主要影响套接字描述符的两个标志是：

  • O_NONBLOCK—非阻塞式I/O；
  • O_ASYNC—信号驱动式I/O；

2.3 设置非阻塞

int flags

if((flags = fcntl(fd, F_GETFL, 0)) < 0)
    err_sys("F_GETFL error");
flags |= O_NONBLOCK;
if (fcntl(fd, F_SETFL, flags) < 0)
    err_sys("F_SETFL error");

不能图方便而直接使用以下代码：

if(fcntl(fd, F_SETFL, O_NONBLOCK) < 0)
    err_sys("F_SETFL error");

这段代码在设置非阻塞标志的同时也清楚了所有其他文件状态标志。

3 几个套接字有用的选项

3.1 SO_REUSEADDR

四个不同的功能：

  • 允许启动一个监听服务器并捆绑其众所周知的端口；
  • 允许在同一端口上启动同一服务器的多个实例；
  • 允许单个进程捆绑同一端口到多个套接字上，只要每次捆绑指定不同的本地IP地址即可；
  • 运行完全重复的捆绑：当一个IP地址和端口已绑定到某个套接字上时，只有UDP支持将同样的IP地址和端口捆绑到另一个套接字上。

使用实例：

Setsockopt(listenfd, SOL_SOCKET, SO_REUSEADDR, &on, sizeof(on));

3.2 SO_RCVBUF和SO_SNDBUF

更改套接字接收缓冲区和发送缓冲区的大小。 使用实例：

int n = 220 * 1024;
Setsockopt(listenfd, SOL_SOCKET, SO_RCVBUF, &on, sizeof(on));

select函数

《Unix网络编程：卷1》``select``posts

第六章主要讲的是I/O复用的2个函数，select和poll函数。现在主要看看select函数

1 函数原型

#include <sys/select.h>
/* According to earlier standards */
#include <sys/time.h>
#include <sys/types.h>
#include <unistd.h>

int select(int nfds, fd_set *readfds, fd_set *writefds,
          fd_set *exceptfds, struct timeval *timeout);

该函数运行进程指示内核等待多个事件中的任何一个发生，并只在有一个或多个时间发生或经历一段指定的时间后才唤醒它。 简单说，使用select就可以实现非阻塞，一旦执行必然会有返回值，以返回值的不同来反映函数的执行情况。

2 参数解析

这几个参数其实都不是很重要

2.1 nfds

是一个整数值，是指集合中所有文件描述符的范围，即所有文件描述符的最大值加1。每次循环，描述符0，1，2，···，一直到nfds-1都会被测试。

怪不得有一个maxfdp1 = max(fileno(fp), sockfd) + 1的语句

2.2 *readfds

指向fd_set结构的指针，监视文件描述符的读变化。如果集合中有一个文件可读，select就会返回一个大于0的值，表示有文件可读。如果没有那么根据timeout参数判断是否超时。超出返回0，错误返回负值。可以传入NULL值，表示不关心任何文件的读变化。

2.3 *writefds

上同，改为写变化。

2.4 *exceptfds

用来观察异常情况。

上述三个参数都值—结果参数。调用select函数时，我们指定所关心的描述符的值，函数返回时，结果将指示哪些描述符已就绪。

2.5 *timeout

告知内核等待所指定描述符中的任何一个就绪可花多长时间。

3 宏定义

有几个宏定义需要了解一下。

3.1 FD_ZERO(fd_set *rset)

初始化文件描述符集合。

3.2 FD_SET(int fd, fd_set *fdset)

将描述符放入集合。

3.3 FD_CLR(int fd, fd_set *fdset)

将描述符移除集合。

3.4 FD_ISSET(int fd, fd_set *fdset)

判断描述符是否在集合中。

4 描述符就绪条件

4.1 套接字准备读

满足以下任一情况则可读：

  • 该套接字接受缓冲区中的数据字节数大于等于套接字接收缓冲区低水位标记的当前大小；
  • 该连接的读半关闭（接收了FIN的TCP连接）；
  • 该套接字是一个监听套接字且已完成的连接数不为0；
  • 其上有一个套接字错误待处理。

4.2 套接字准备写

满足以下任一情况则可写：

  • 该套接字发送缓冲区中的可用空间字节数大于等于套接字发送缓冲区低水位标记的当前大小，并且或者该套接字已连接，或者该套接字不需要连接（如UDP套接字）；
  • 该连接的写半部关闭，对这样的套接字进行写操作会产生SIGPIPE信号；
  • 使用非阻塞式connect的套接字已建立连接；
  • 其上只有一个套接字错误待处理。

处理SIGCHLD信号

《Unix网络编程：卷1》``posts 为了防止fork的子进程编程僵死进程，需要对其调用wait函数。那么需要建立一个俘获SIGCHLD信号的信号处理函数，在函数体中调用wait。

SIGCHLD，在一个进程终止或者停止时，将SIGCHLD信号发送给其父进程，按系统默认将忽略此信号，如果父进程希望被告知其子系统的这种状态，则应捕捉此信号。

1 信号函数

void sig_chld(int signo)
{
    pid_t   pid;
    int     stat;

    pid = wait(&stat);
    printf("child %d terminated\n", pid);
    return;
}


//在main函数里绑定
signal(SIGCHLD, sig_chld);

2 wait函数

pid_t wait(int *status);

进程一旦调用了wait，就立即阻塞自己，由wait自动分析是否当前进程的某个子进程已经退出，如果让它找到了这样一个已经变成僵尸的子进程，wait就会收集这个子进程的信息，并把它彻底销毁后返回；如果没有找到这样一个子进程，wait就会一直阻塞在这里，直到有一个出现为止。

2.1 参数status

一开始一直不懂这个参数是干什么的。 status用于保存出发wait的信号值或者退出时exit(code)中的code值。 wait.h中定义了一些宏用于解析status的值：

宏

含义

WIFEXITED(status)

子进程正常退出返回true否则false

WEXITSTATUS(status)

当正常退出时，返回exit(code)中的code

WIFSIGNALED

子进程接受信号退出时返回true，否则false

WTERMSIG

被信号量杀死时，返回信号量的值

WIFSTOPED(status)

当子进程被信号量暂停时返回true

WSTOPSIG(status)

被信号量暂停时信号量的值

3 wiatpid函数

3.1 函数原型

pid_t waitpid(pid_t pid, int *status, int options);

尽量用这个函数，而不要用wait函数。 waitpid函数就等待哪个进程以及是阻塞给了我们更多的控制。首先，pid参数运行我们指定想等待的进程ID，值-1表示等待第一个终止的子进程。其次，options参数允许我们指定附加选项。最常用的选项是WNOHANG，它告知内核在没有已终止子进程时不要阻塞。 例子：

void
sig_chld(int signo)
{
    pid_t   pid;
    int     stat;

    while ( (pid = waitpid(-1, &stat, WNOHANG)) > 0)
        printf("child %d terminated\n", pid);
    return;
}

3.2 options值:

常量

含义

WNOHANG

调用wait时制定pid仍未返回，wait立即返回0，用于判断子进程有没有结束

WUNTRACED

当子进程被暂停时，则wait立即返回子进程的pid

WCONTINUED

当被暂停的子进程又被信号量恢复后，则wait立即返回子进程的pid。

基本套接字编程

《Unix网络编程：卷1》``posts

这一章大部分函数在之前的笔记已经有记录，主要看一下fork函数

1 fork函数

之前有接触过这个函数，但是理解起来还是有点难。

1.1 函数原型

#include <unistd.h>
pit_t fork(void);

返回：在子进程中为0，在父进程中为子进程ID，若出错则为-1

1.2 返回2个值

fork在子进程返回0而不是父进程的进程ID的原因在于： 任何子进程只有一个父进程，而且子进程总是可以通过调用getppid取得父进程的进程ID。相反，父进程可以有许多子进程，而且无法获取各个子进程的进程ID。如果父进程想要跟踪子进程的进程ID，那么它必须记录每次调用fork的返回值。

1.3 两个用法

1） 处理各自操作

一个进程创建另一个自身的副本，自己执行自己的操作。一般是网络服务器的典型用法。

2）执行另一个程序

就是把子进程升级为主进程，fork之后调用exec把自身替换成新的程序，shell一般这样做。

2 exec函数

把当前进程映像替换成新的程序文件，进程ID并不改变，但是内容变了。

2.1 函数原型

#include <unistd.h>
       extern char **environ;
       int execl(const char *path, const char *arg, ...
                       /* (char  *) NULL */);
       int execlp(const char *file, const char *arg, ...
                       /* (char  *) NULL */);
       int execle(const char *path, const char *arg, ...
                       /*, (char *) NULL, char * const envp[] */);
       int execv(const char *path, char *const argv[]);
       int execvp(const char *file, char *const argv[]);
       int execvpe(const char *file, char *const argv[],
                       char *const envp[]);

2.2 区别

这6个函数之间的区别在于：

  • 待执行的程序文件是由文件名还是路径指定；
  • 新程序的参数是一一列出还是由一个指针数组来引用；
  • 把调用进程的环境传递给新程序还是给新程序指定新的环境。

3 并发服务器

Unix中编写并发服务器程序最简单的办法就是fork一个子进程来服务每个客户。

3.1 引用计数

这是个比较新的概念

在文件表中还有一项：打开文件引用计数，引用这个文件对象的描述符数（dup和fork都会增加这个计数，第一次open会使此计数为1，close会减小此计数，为0时销毁文件对象）。所以在程序的运行过程中，不管是父进程还是子进程先close(fd)或fclose(fp),该文件表依然存在并没有被销毁，只有当文件表中的打开文件引用计数为0时才会销毁这个文件表对象。

为什么调用close没有终止与客户的连接？

每个文件或套接字都有一个引用计数，它是当前打开着的引用该文件或套接字的描述符的个数。socket返回后与listenfd关联的文件表象的引用计数值为1。accept返回后与connfd关联的文件表项的引用计数值也为1。然而fork返回后，这两个描述符就在父进程与子进程间共享，因此与这两个套接字相关联的文件表项各自访问计数值均为2。所以当父进程关闭connfd时，它只是把相应的引用计数值从2减为1.该套接字真正的清理和资源释放要等要其引用计数值到达0时才发生。

3.2 关闭连接

因为fork会复制主进程的所有东西，但是子进程不需要监听套接字所以就在子进程里关闭了监听套接字，但是这并没有影响主进程的监听套接字。 主进程也不需要处理多出来的已连接套接字，同样将其关闭。

套接字编程简介

《Unix网络编程：卷1》``SOCKET``posts

这一章主要是介绍一下套接字socket

地址结构就不再介绍了，之前有介绍IPv4的结构体，主要是要注意一下sockaddr是通用套接字地址结构。

1 值—结果参数

有趣的概念，套接字地址结构传递方式取决于该结构的传递方向。

1.1 进程→内核

有三个函数是从进程到内核传递套接字地址结构：

  • bind
  • connect
  • sendto

它们传递结构体的时候都是引用方式，因为要改变原本地址的值，不能按值传递。

1.2 内核→进程

有四个函数是从内核到进程传递套接字地址结构：

  • accept
  • recvfrom
  • getsockname
  • getpeername

要注意这里传递的时候和上一个不一样，传递长度的时候也是引用。其原因在与： 当函数被调用时，结构大小是一个值，它告诉内核该结构的大小，这样内核在写该结构时不至于越界；当函数返回时，结构大小又是一个结果，它告诉进程内核在该结构究竟存储了多少信息。这种类型的的参数称为值—结果参数。

int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);

可以看到accept函数第三个参数也是指针类型，所以结构体长度可能会改变。

2 字节排序函数

这里主要介绍一下大端小端的概念。

2.1 大端

将高序字节存储在起始地址

2.2 小端

将低序字节存储在起始地址

2.3几个转换函数

#include <netinet/in.h>
uint16_t htons(uint16_t host16bitvalue)
uint32_t htonl(uint32_t host32bitvalue)
uint16_t ntohs(uint16_t host16bitvalue)
uint32_t ntohl(uint32_t host32bitvalue)

h代表host，n代表network，s代表short，l代表long。 前两个函数的作用是：将主机字节序转换为网络字节序；后两个则相反。

3 几个socket会用到的函数

3.1 bzero函数

函数原型：

#include <strings.h>
void bzero(void *s, size_t n);

主要是把目标字节串中指定数据的字节置为0，和memset的区别是少了一个参数，memset可以指定要置的值，如果只需要置为0那么可以用bzero函数

3.2 inet_pton函数

函数原型：

#include <arpa/inet.h>
int inet_pton(int af, const char *src, void *dst);

af参数为AF_INET或者AF_INET6，前面有说过是什么。这个函数的功能是将“点分十进制” －> “二进制整数”，将字符串转换为二进制网络地址。

TCP连接的建立和终止

《Unix网络编程：卷1》``TCP``posts

初识Linux中的socket

《Unix网络编程：卷1》``posts

和之前的QT中的socket大有不同，C语言中的socket好像要复杂得多。

1 一切都是文件

经过前面的学习，已经知道在Linux中一切皆为文件，那么文件就会有文件标识符。 所以我们可以看到在实际编程中socket和连接等的文件标识符都是int类型。 比如：

int listenfd, connfd;
listenfd = socket(AF_INET, SOCK_STREAM, 0 );
//socket 会返回一个int类型的描述符
.....

2 socket

socket函数创建一个网际字节流套接字，它是TCP套接字的花哨名字。该函数返回一个小整数描述符，以后的所有函数调用（如随后的bind和listen）就用该描述符标识这个套接字。

2.1 函数原型

int socket(int domain, int type, int protocol);
//返回负数则创建失败，大于0成功

2.2 参数详解

  • 第一个参数都是AF_INET 其中AF 表示ADDRESS FAMILY地址族，总的意思是IPv4 Internet protocols表示IPV4地址族。IPv6则用AF_INET6。
  • 第二个参数type，一般是SOCK_STREAM 表示提供面向连接的稳定数据传输，即TCP协议。
  • 第三个参数protocol，一般都是0 指定协议，常用协议有IPPROTO_TCP、IPPROTO_UDP等，type和protocol不能随意组合，要选择type对应的protocol。当参数为0时，会自动选择type对应的默认协议。

2.3 实际使用

int listenfd = socket(AF_INET, SOCK_STREAM, 0 );

服务端

标准的TCP服务器准备监听描述符的步骤：

  • 创建socket
  • bind地址
  • listen监听

3 bind

无论是服务器还是客户端，都需要创建socket，在服务器的一端需要将socket绑定端口和IP。（这样的话还要先介绍一下sockaddr和sockaddr_in，反正记住它是一个地址结构体，用来存放地址和端口等信息）

3.1 函数原型：

int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
//返回：0 ──成功， - 1 ──失败

3.2 参数详解

  • 第一个参数sockfd，这个在上一节有说到，就是socket的文件描述符。

  • 第二个参数*addr，可以看到前面是const struct sockaddr类型，说明是个常量结构体。一般新建结构体的时候是用sockaddr_in，在用的时候再将其强制转换为sockaddr类型。

    struct sockaddr_in {
    short int sin_family; /* Address family /
    unsigned short int sin_port; /     Port number /
    struct in_addr sin_addr; /     Internet address /
    unsigned char sin_zero[8]; /     Same size as struct sockaddr */
    };

  • 第三个参数addrlen，为结构体的长度，一般用sizeof来求的。

3.3 实际使用

函数调用的例子：

struct sockaddr_in saddr； 
saddr.sin_family = AF_INET; 
saddr.sin_port = htons(8888); //端口
saddr.sin_addr.s_addr = htonl(INADDR_ANY);//监听任意地址 
bind(ListenSocket,(struct sockaddr *)&saddr,sizeof(saddr))； //绑定

和QT有不同的地方是，QT里好像没有bind过程，直接监听了。然后监听的时候指定了端口和IP。

4 listen

调用listen函数把该套接字转换成一个监听套接字，这样来自客户的外来连接就可在该套接字上由内核接受。

4.1 函数原型

int listen(int sockfd, int backlog);
//返回：0──成功， -1──失败

4.2 参数详解

  • 第一个参数sockfd，套接字描述符。
  • 第二个参数backlog，这个参数涉及到一些网络的细节，主要是限制监听描述符上排队的最大客户连接数。

4.3 实际使用

listen(listenfd, LISTENQ);  //这个LISTENQ为自定义的

4.4 backlog

内核为任何一个给定的监听套接字维护两个队列：

  • 未完成连接队列，处于SYN_RCVD状态的套接字；
  • 已完成连接队列，处于ESTABLISHED状态的套接字。

以上两队列之和不超过backlog

5 accept

accept()系统调用主要用在基于连接的套接字类型，比如SOCK_STREAM和SOCK_SEQPACKET。它提取出所监听套接字的等待连接队列中第一个连接请求，创建一个新的套接字，并返回指向该套接字的文件描述符。新建立的套接字不在监听状态，原来所监听的套接字也不受该系统调用的影响。

之前QT里写的，里面还有分Server这个类，专门用来处理连接，incomingconnection函数处理客户端请求，然后将socketId发给新的socket用来处理数据。

区分这两个套接字非常重要。一个服务器通常仅仅创建一个监听套接字，它在该服务器的生命期内一直存在。

5.1 函数原型

int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);

最多返回三个值：

  • 新套接字描述符或者出错指示的值
  • 客户进程的协议地址（由addr所指）
  • 该地址的大小（addrlen所指）

并不是说返回三个int，而是有三个值因为这个函数而改变。

5.2 参数详解

  • 第一个参数sockfd，在这里要注意，用的是监听sockfd，然后会返回一个新的sockfd，用来发送数据接收数据等。
  • 第二个参数*addr，一样，客户端地址。
  • 第三个参数*addrlen，结构体长度。

如果我们对客户端的协议地址不感兴趣，可以把arrd和addrlen均置为空指针。

5.3 实际使用

int connfd = accept(listenfd, (sockadr *)NULL, NULL);

客户端

6 connect

通常用于客户端建立TCP连接。创建socket的步骤就省略了，服务端和客户端创建socket的方法一样。

6.1 函数原型

int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
//返回负数则连接失败

6.2 参数详解

  • 第一个参数socket，不说了。
  • 第二个参数*addr，在这里是socket想连接的地址和端口，结构体设置步骤和3.3节一样。
  • 第三个参数addrlen，结构体长度。

6.3 实际使用

struct sockaddr_in servaddr;
.....

if (connect(sockfd, (sockadd *) &servaddr, sizeof(servaddr)) < 0)
        //错误信息

通信

主要介绍一下读和写

7 write

将缓冲区的数据写入套接字

7.1 函数原型

ssize_t write(int fd, const void *buf, size_t count);
//返回：若成功则返回写入的字节数，若出错则返回-1

其实这个函数很多地方都有用到，不止是socket才用。但是原理一样，将数据写入sockfd。

7.2 参数详解

  • 第一个参数fd，在这里可以理解为sockfd，套接字的文件描述符；
  • 第二个参数*buf，缓冲区，一般是字符缓冲区。用来存放数据；
  • 第三个参数count，缓冲区长度，在写入的时候要指明写入多少数据。

7.3 实际使用

char buff[1024];

write(connfd, buff, strlen(buff));
//connfd来自上一节

8 read

从文件读数据，在这里是从套接字中读数据。使用方法和write一样。

8.1 函数原型

ssize_t read(int fd, void *buf, size_t count);
// 返回：若成功则返回读到的字节数，若已到文件末尾则返回0，若出错则返回-1

8.2 参数详解

  • 第一个参数fd，套接字文件描述符
  • 第二个参数*buf，缓冲区，一般是字符缓冲区。用来存放数据；
  • 第三个参数count，读入多少数据。

8.3 实际使用

char recvline[MAXLINE];
while(( n = read(sockfd, recvline, MAXLINE)) > 0)
....
