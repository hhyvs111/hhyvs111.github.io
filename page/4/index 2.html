标准库类型vector表示对象的集合，其中所有对象的类型都相同。集合中的每个对象都有一个与之对应的索引，用于访问对象。一般称之为容器。

说明

vector v1

v1是一个空vector,它潜在的元素是T类型的，执行默认初始化

vector v2 (v1)

v2中包含有v1所有元素的副本.

vector v2 = v1

等价于v2(v1)，v2中包含有v1所有元素的副本.

vector v3 (n, val)

v3包含了n个重复的元素，每个元素的值都是val

vector v4 (n)

v4包含了n个重复地执行了值初始化的对象

vector v5{a,b,c…}

v5 包含了初始值个数的元素，每个元素被赋予相应的初始值

vector v5={a,b,c…}

等价于 v5{a,b,c…}

这个重复元素可以关注一下，可以利用这个来初始化。

默认初始化

如果是内置类型，比如int，则元素初始值自动设为0，是string则是空。

vector<int> ivec(10) ; // 10个元素，每个都初始化为0
vector<string> svec(10) ; // 10个元素，每个都是空string对象

这里要特别小心区别花括号和圆括号，效果不同。

vector<int> v1 (10) ;
// v1有10个元素，每个的值都是0
vector<int> v2{10} ;
// v2有1个元素，该元素的值是10
vector<int> v3(10， 1) ;
// v3有10个元素，每个的值都是1
vector<int> v4{10，1} ;
//v4有2个元素，值分别是10和1

添加元素

这里push_back()应该比较熟悉，一般如果数组不能确定大小，那么就用vector。

Ubuntu MySQL 远程连接错误2003 Can't connect to MySQL server on (61 "Connection refused")

Posted on 2018-12-25 Edited on 2020-04-14 In Linux , 教程

远程连接服务器上的mysql出现错误，后面查阅了一些资料，但是大部分都写的不是很齐全。如果你用的是Ubuntu系统，那么首先做这两件事，就能确保连接了。

MySQL设置

首先确保你要连接的账号支持远程登录，可以在mysql数据库下查看用户的登录权限，比如root。

mysql -u root -p 
#输入密码
mysql>use mysql;
mysql>select host,user from user;

￼

如果你的root的host是localhost，那么是连接不上的。需要改成%。

update user set host = '%' where user = "root";

检查MySQL服务

还要检查一下这个MySQL的监听服务是不是开了。

#假如你的数据库端口是3306
netstat -an | grep 3306

￼
这里如果前面的是127.0.0.1的话那么只能监听此ip，不接受别的ip访问。

修改配置文件

Ubuntu的是在/etc/mysql/mysql.conf.d/mysqld.cnf

找到里面的bind-address，改成0.0.0.0或者你想要的ip。

测试一下即可连接成功。

String

这个东西需要记一下。

定义和初始化

直接初始化和拷贝初始化

简单来说有=号就是拷贝初始化 当初始值只有一个时，使用直接初始化或拷贝初始化都行。如果像上面的s4那样初始化要用到的值有多个，一般来说只能使用直接初始化的方式:

string s5 = "hiya";     //拷贝初始化
string s6 ("hiya") ;    //直接初始化
string s7 (10，'c') ;    //直接初始化，s7的内容是cccccccccc

对于用多个值进行初始化的情况，非要用拷贝初始化的方式来处理也不是不可以，不过需要显式地创建一个(临时)对象用于拷贝:

string s8 = string(10，'c') ; //拷贝初始化，s8的内容是CCcccccccc

s8的初始值是string(10, ‘c’)，它实际上是用数字10和字符c两个参数创建出来的一个string对象，然后这个string对象又拷贝给了s8。这条语句本质上等价于下面的两条语句:

string temp(10， ' C') ;
// temp的内容是cccccccccc
string s8 = temp;
//将temp拷贝给s8

一般都用括号直接初始化比较好。

比较大小

String遵循以下两条规则：

  1. 如果两个string对象的长度不同，而且较短string对象的每个字符都与较长string对象对应位置上的字符相同，就说较短string对象小于较长string对象。

  2. 如果两个string对象在某些对应的位置上不一致，则string对象比较的结果其实是string对象中第一对相异字符比较的结果。

下面是string对象比较的一个示例:

string str = "Hello";
string phrase = "Hello World";
string slang = "Hiya";

根据规则1可判断，对象str小于对象phrase;根据规则2可判断，对象slang既大于str也大于phrase（因为i大于e）。

字面值和string对象相加

有时候会用到这个string和char字符相加，这个时候要确保+号的两侧的运算对象至少有一个是string。

strings4=s1+"，";    //正确:把一个string对象和一个字面值相加
string s5 = "hello" +”，";   //错误:两个运算对象都不是string
//正确:每个加法运算符都有一个运算对象是string
string s6 = s1 +",”+ "world";
string s7 = "hello" + ",”+s2; //错误:不能把字面值直接相加

空指针

Posted on 2018-12-19 Edited on 2020-04-14 In C++

空指针 空指针(null pointer)不指向任何对象，在试图使用个指针之前代码可以首先检查它是否为空。以下列出几个生成空指针的方法:

int *pl = nullptr;
//等价于int*p1=0;
int *p2 = 0;
//直接将p2初始化为字面常量0
//需要首先#include cstdlib
int *p3 = NULL;
//等价于int*p3=0;

得到空指针最直接的办法就是用字面值nullptr来初始化指针,这也是C++11新标准刚刚引入的一种方法。nullptr是一种特殊类型的字面值，它可以被转换成任意的其他指针类型。

牛客网刷题之前也有看到这个nullptr，还以为是错的。还有这个NULL，要引用头文件才能用。

当用到一个预处理变量时，预处理器会自动地将它替换为实际值，因此用NULL初始化指针和用0初始化指针是一样的。在新标准下，现在的C++程序最好使用nullptr,同时尽量避免使用NULL。

初始化所有指针

建议:初始化所有指针 使用未经初始化的指针是引发运行时错误的一大原因。和其他变量一样，访问未经初始化的指针所引发的后果也是无法预计的。通常这一行为将造成程序崩溃，而且一旦崩溃，要想定位到出错位置将是特别棘手的问题。 在大多数编译器环境下，如果使用了未经初始化的指针，则该指针所占内存空间的当前内容将被看作一个地址值。访问该指针，相当于去访问一个本不存在的位置上的本不存在的对象。糟糕的是，如果指针所占内存空间中恰好有内容，而这些内容又被当作了某个地址，我们就很难分清它到底是合法的还是非法的了。 因此建议初始化所有的指针，并且在可能的情况下，尽量等定义了对象之后再定义指向它的指针。如果实在不清楚指针应该指向何处，就把它初始化为nullptr或者0,这样程序就能检测并知道它没有指向任何具体的对象了。

简单来说就是指针初始化都直接指向nullptr吧。

1. Two Sum

Posted on 2018-11-26 Edited on 2020-04-14 In 算法

要开始刷算法题了，昨天去头条笔试被吊打了，现在的自己原来这么菜。

那么就从LeetCode开始刷吧。

题目描述

Given an array of integers, return indices of the two numbers such that they add up to a specific target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.
Example:

Given nums = [2, 7, 11, 15], target = 9,
Because nums[0] + nums[1] = 2 + 7 = 9,
return [0, 1].

思路

就是在数组里找两个数是否是相加等于这个目标值。

直接想到的方法就是遍历，然后判断是否相等。

代码

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        vector<int> ans;
        int size = nums.size();
        for(int i = 0; i < size; i++){
            for(int j = i + 1 ;j < size; j++){
                if(nums[i] + nums[j] == target){
                    ans.push_back(i);
                    ans.push_back(j);
                    return ans;
                }
            }
        }
    }
};

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
      std::unordered_map<int, std::size_t> tmp;  
      for (std::size_t i = 0; i < nums.size(); ++i) {
        if (tmp.count(target - nums[i])) {
            return {tmp[target - nums[i]], i};
        }
        tmp[nums[i]] = i;
    }
        return {0,0};

    }
};

// The number type to use for tessellation
using Coord = double;

// The index type. Defaults to uint32_t, but you can also pass uint16_t if you know that your
// data won't have more than 65536 vertices.
using N = uint32_t;

// Create array
using Point = std::array<Coord, 2>;
std::vector<std::vector<Point>> polygon;

// Fill polygon structure with actual data. Any winding order works.
// The first polyline defines the main polygon.
polygon.push_back({ {100, 0}, {100, 100}, {0, 100}, {0, 0}});
// Following polylines define holes.
polygon.push_back({ {75, 25}, {75, 75}, {25, 75}, {25, 25}});

// Run tessellation
// Returns array of indices that refer to the vertices of the input polygon.
// e.g: the index 6 would refer to {25, 75} in this example.
// Three subsequent indices form a triangle. Output triangles are clockwise.
std::vector<N> indices = mapbox::earcut<N>(polygon);

获得三角形

这里要注意，上述代码输入了2个多边形，那么再获取相关剖分数据的时候要区别一下。

主要是获取这个各个点的连接顺序，也就是三角剖分。indices的值就是点的顺序。比如indices的值是24，那么就是有8个三角形（上面有2个正方形组成的带孔的正方形）。按indices输出顺序连接三角形，就是剖分结果了。

    for (const auto& i : indices) {
        Point point;
        cout  << i<< endl;
//        cout << indices.at(i) << endl;
//        cout << endl;
        //如果大于4则是访问第二个多边形
        if(i >= 4)
        {
            //
            point.x = polygon.at(1).at(i - 4).at(0);
            point.y = polygon.at(1).at(i - 4).at(1);
        }
        else
        {
            point.x = polygon.at(0).at(i).at(0);
            point.y = polygon.at(0).at(i).at(1);
        }

        cout <<  point.x<< " " << point.y << endl;
    }

这样就能正确的获取三角形的连接顺序，然后就正常操作就好了。

glfwGetKey多次触发的问题

最近写程序的时候用到了glfwGetKey，在里面写了一些相关的触发函数。但是奇怪的发现，我只按了一次触发键，却引起了多次调用。不知道是为什么，一开始没发现导致我踩了很多坑。

if (glfwGetKey(window, GLFW_KEY_F1) == GLFW_PRESS)
{
    function();
}

比如说你绑定了F1为某个功能键，你会发现你按了一下，却多次触发了function。目前原因不明，我的解决方法是加一个全局变量开关。

if (glfwGetKey(window, GLFW_KEY_F1) == GLFW_PRESS)
{
    if(!functionOpen)
    {
        function();
        functionOpen = true;
    }
}

不是完美的解决方法但是还是可以用的。
