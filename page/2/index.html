101. Symmetric Tree 镜像二叉树

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

判断一颗二叉树是否是镜像二叉树，也就是中心对称。

For example, this binary tree [1,2,2,3,4,4,3] is symmetric:

    1
   / \
  2   2
 / \ / \
3  4 4  3
But the following [1,2,2,null,3,null,3] is not:
    1
   / \
  2   2
   \   \
   3    3

题解

递归版

利用递归思想, root相等且递归t1的左节点等于t2的右节点相等，那么才返回true，如果一个为空，另一个不为空直接返回false。

class Solution {
public:
    bool isSymmetric(TreeNode* root) {
        return mirror(root,root);
    }
    bool mirror(TreeNode* t1, TreeNode* t2){
        if(t1 == NULL && t2 == NULL) return true;
        if(t1 == NULL || t2 == NULL) return false;
        return (t1->val == t2->val) && mirror(t1->left,t2->right) && mirror(t1->right, t2->left);
    }
};

迭代版

用栈做，push两次根节点，然后取出来判断。再依次push进去t1的左节点和t2的右节点，因为相近的就是要比较的，每次弹出两个来比较。

class Solution {
public:
    vector<int> nums;
    vector<int> numsRev;
    bool isSymmetric(TreeNode* root) {
        if(root == NULL)
            return true;
        //左右一个栈，同时进行递归
        //镜面原理，将一个树分为两颗子树，他们符合一个规律。左子树等于另一个树的右子树。
        stack<TreeNode*> s;
        s.push(root);
        s.push(root);
        while(!s.empty())
        {
            TreeNode* t1 = s.top();
            s.pop();
            TreeNode* t2 = s.top();
            s.pop();
            //如果都为空则继续，可能有空节点
            if(t1 == NULL && t2 == NULL) continue;
            if(t1 == NULL || t2 == NULL) return false;
            if(t1->val != t2->val) return false;
            s.push(t1->left);
            s.push(t2->right);
            s.push(t1->right);
            s.push(t2->left);
        }
        return true;
    }
};

53. Maximum Subarray 最大的连续子数组和

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

给一串数字，求出连续的子数组的最大和。用O(n)解决。

Input: [-2,1,-3,4,-1,2,1,-5,4],
Output: 6
Explanation: [4,-1,2,1] has the largest sum = 6.

题解

依次累加，定义一个当前sum和最大sum。

  • 如果nowSum加了这个值是大于nums[i]的，那么就加，因为是正向的；
  • 如果nowSum加了这个值比当前的值还要小，那么这个序列是成负向的，直接把标号移动到nums[i]的位置。

因为要尽量跳过负数，所以如果减到0那么就要重新累加。

class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int nowSum = nums[0];
        int maxSum = nowSum;
        for(int i = 1; i < nums.size();i++)
        {
            nowSum = (nowSum + nums[i])  >= nums[i]? (nowSum + nums[i]) : nums[i];
            maxSum = max(maxSum,nowSum);
        }
        return maxSum;
    }
};

70. Climbing Stairs 爬楼梯

发表于 2019-03-22 更新于 2020-04-14 分类于 C++

题意

给你一个高度为n的台阶，每次你可以爬1层台阶，或者爬2层台阶。有多少种爬楼梯的方法。

Example 1:
Input: 2
Output: 2
Explanation: There are two ways to climb to the top.
1. 1 step + 1 step
2. 2 steps

Example 2:
Input: 3
Output: 3
Explanation: There are three ways to climb to the top.
1. 1 step + 1 step + 1 step
2. 1 step + 2 steps
3. 2 steps + 1 step

题解

动态规划，初始化1为1种，2为两种。dp[i]为前面两个dp的和。

class Solution {
public:
    int climbStairs(int n) {
        //据说是最简单的dp？
        if(n == 1) return 1;
        vector<int> dp (n+1);
        dp[1] = 1;
        dp[2] = 2;
        //从前面推后面
        for(int i = 3 ; i <= n;i++)
        {
            dp[i] = dp[i-1] + dp[i-2];

        }
        return dp[n];
    }
};

121. Best Time to Buy and Sell Stock 股票最佳买卖 — 单次买卖

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

一个数组，代表股票当前价格，买入和买出。怎么操作能使收益最大？只有卖掉当前股票才能买。

Example 1:
Input: [7,1,5,3,6,4]
Output: 5
Explanation: Buy on day 2 (price = 1) and sell on day 5 (price = 6), profit = 6-1 = 5.
             Not 7-1 = 6, as selling price needs to be larger than buying price.

Example 2:
Input: [7,6,4,3,1]
Output: 0
Explanation: In this case, no transaction is done, i.e. max profit = 0.

题解

菜鸡版

因为只能买卖一次，求出最大的值，然后求出最小的值。当最小值出现的时候，如果最大值的下标小于最小值的下标，那么重置最大值

class Solution {
public:
    int maxProfit(vector<int>& prices) {
        if(prices.size() <= 1) return 0;
        int maxProfit = 0;
        int minPrice = prices[0], maxPrice = 0;
        int indexMax = 0, indexMin = 0;
        for(int i = 0;i < prices.size();i++)
        {
            if(prices[i] > maxPrice)
            {
                maxPrice = prices[i];
                indexMax = i;
            }
            if(prices[i] < minPrice)
            {
                minPrice = prices[i];
                indexMin = i;
                //当最小值出现的时候，如果最大值的下标小于最小值的下标，那么重置最大值
                if(indexMin > indexMax)
                    maxPrice = 0;
            }
            if(maxPrice - minPrice > maxProfit)
                maxProfit = maxPrice - minPrice;
        }
        return maxProfit;
    }
};

大佬版

遍历一次数组，用一个变量记录遍历过数中的最小值，然后每次计算当前值和和这个最小值之间的差值作为利润，然后每次选较大的利润来更新，当遍历完成后当前利润即为所求。

class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int res = 0, buy = INT_MAX;
        for (int price : prices) {
            buy = min(buy, price);
            res = max(res, price - buy);
        }
        return res;
    }
};

448. Find All Numbers Disappeared in an Array 找出数组里消失的值

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

给你一个数组，其值是从1到n（数组长度）的数，但是有一些数没有出现，让你找出这些数。

Input:
[4,3,2,7,8,2,3,1]

Output:
[5,6]

题解

比如这里5，6没有出现，但是2和3重复出现了两次。可以利用这个性质求解，将数字放到数组中正确位置。然后再循环一遍数组，当数组的位置与值不对应，就是要找的值。

空间复杂度O(1)

class Solution {
public:
    vector<int> findDisappearedNumbers(vector<int>& nums) {
        vector<int> ans;
        for(int i = 0 ; i < nums.size();i++)
        {
            //将数组放回原来的位置
            if(nums[i] != nums[nums[i] - 1])
            {
                swap(nums[i], nums[nums[i] - 1]);
                --i;
            }
        }
        for(int i = 0 ; i < nums.size();i++)
        {
            if(nums[i] != i + 1)
                ans.push_back(i+1);
        }
        return ans;
    }
};

283. Move Zeroes 移动0

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

给一串数组，将里面所有的0都移动到后面，而且不能改变原来非0数字的前后顺序。原地交换，不能用额外的空间。

Input: [0,1,0,3,12]
Output: [1,3,12,0,0]

题解

用一个计数器，每次遇到非0就把计数器对应数组的位置的值置为非0值，然后再用0填充后面的数字。

class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        //找到0则互换，必须原地互换
        int n = nums.size() - 1;
        int cnt = 0;
        for(int i = 0;i <= n;i++)
        {
            if(nums[i] != 0)
            {
                nums[cnt++] = nums[i];
            }
        }
        for(int i = cnt; i <= n;i++)
            nums[i] = 0;
    }
};

226. Invert Binary Tree 反转二叉树

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

反转二叉树。

Input:

     4
   /   \
  2     7
 / \   / \
1   3 6   9
Output:

     4
   /   \
  7     2
 / \   / \
9   6 3   1

题解

比较经典的一道题。

递归版

先定义左右节点等于递归后的左右节点，然后交换他们的值。

class Solution {
public:
    TreeNode* invertTree(TreeNode* root) {
        if(!root) return NULL;
        TreeNode* right = invertTree(root->right);
        TreeNode* left = invertTree(root->left);
        root->left = right;
        root->right = left;

        return root;
    }
};

迭代版

利用栈，先把左右子树存进去，然后交换。

TreeNode* invertTree(TreeNode* root) {
    std::stack<TreeNode*> stk;
    stk.push(root);

    while (!stk.empty()) {
        TreeNode* p = stk.top();
        stk.pop();
        if (p) {
            stk.push(p->left);
            stk.push(p->right);
            std::swap(p->left, p->right);
        }
    }
    return root;
}

104. Maximum Depth of Binary Tree 二叉树的最大深度

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

求二叉树的最大深度

  3
 / \
9  20
  /  \
 15   7

输出3。

题解

复杂递归

用一个全局变量保存最大值，当可以继续递归的时候加1，然后判断是否比max大。

class Solution {
public:
    int max = 0;
    int maxDepth(TreeNode* root) {
        if(!root) return 0 ;
        dfs(root, 0);
        return max + 1;
    }
    void dfs(TreeNode* root, int now)
    {
        if(!root) return;
        if(root->left) 
        {
            dfs(root->left, now + 1);
        }
        if(root->right)
        {
            dfs(root->right, now + 1);
        }
        if(now > max)
            max = now;
    }
};

简洁递归

每次都会比较左右子树，然后累计。

class Solution {
public:
    int maxDepth(TreeNode* root) {
        //用dfs
        if(root == NULL)
            return 0;
        return 1 + max(maxDepth(root->left), maxDepth(root->right));
    }
};

617. Merge Two Binary Trees 合并二叉树

发表于 2019-03-22 更新于 2020-04-14 分类于 算法

题意

两个二叉树，将其合并并输出根节点。递归的写法很简洁，可以学习一下。

Input: 
    Tree 1                     Tree 2                  
          1                         2                             
         / \                       / \                            
        3   2                     1   3                        
       /                           \   \                      
      5                             4   7                  
Output: 
Merged tree:
         3
        / \
       4   5
      / \   \ 
     5   4   7

题解

直接相加就好了，如果不存在则返回另一个。

递归求解

class Solution {
public:
    TreeNode* mergeTrees(TreeNode* t1, TreeNode* t2) {
        //递归做，如果都存在则相加，一个不存在则返回另一个存在的。
        if(t1 == NULL)
            return t2;
        if(t2 == NULL)
            return t1;
        t1->val += t2->val;
        t1->left = mergeTrees(t1->left, t2->left);
        t1->right = mergeTrees(t1->right, t2->right);
        return t1;
    }
};

迭代解法Java版

public class Solution {
    public TreeNode mergeTrees(TreeNode t1, TreeNode t2) {
        if (t1 == null)
            return t2;
        Stack < TreeNode[] > stack = new Stack < > ();
        stack.push(new TreeNode[] {t1, t2});
        while (!stack.isEmpty()) {
            TreeNode[] t = stack.pop();
            if (t[0] == null || t[1] == null) {
                continue;
            }
            t[0].val += t[1].val;
            if (t[0].left == null) {
                t[0].left = t[1].left;
            } else {
                stack.push(new TreeNode[] {t[0].left, t[1].left});
            }
            if (t[0].right == null) {
                t[0].right = t[1].right;
            } else {
                stack.push(new TreeNode[] {t[0].right, t[1].right});
            }
        }
        return t1;
    }
}

单线程服务器的常用编程模型

一般比较广泛的就是（非阻塞IO多路复用和）“non-blocking IO + IO multiplexing”，也就是Reactor模式。

定义

在非阻塞IO多路复用里，程序的基本结构是一个事件循环，以事件驱动和事件回调的方式实现业务逻辑。

建议在Linux里使用epoll。

基于事件驱动的编程模型也有其本质的缺点，它要求事件回调函数必须是非阻塞的。对于涉及网络IO的请求响应式协议，它容易割裂业务逻辑，使其散步于多个回调函数之中，相对不容易理解和维护。

Reactor模型

Reactor模式是处理并发I/O比较常见的一种模式，用于同步I/O，中心思想是将所有要处理的I/O事件注册到一个中心I/O多路复用器上，同时主线程/进程阻塞在多路复用器上；一旦有I/O事件到来或是准备就绪(文件描述符或socket可读、写)，多路复用器返回并将事先注册的相应I/O事件分发到对应的处理器中。

Reactor是一种事件驱动机制，和普通函数调用的不同之处在于：应用程序不是主动的调用某个API完成处理，而是恰恰相反，Reactor逆置了事件处理流程，应用程序需要提供相应的接口并注册到Reactor上，如果相应的事件发生，Reactor将主动调用应用程序注册的接口，这些接口又称为“回调函数”。用“好莱坞原则”来形容Reactor再合适不过了：不要打电话给我们，我们会打电话通知你。

Reactor模式与Observer模式在某些方面极为相似：当一个主体发生改变时，所有依属体都得到通知。不过，观察者模式与单个事件源关联，而反应器模式则与多个事件源关联。

多线程服务器的常用编程模型

此书选择的是非阻塞IO多路复用和one loop per thread方式来编写多线程C++网络服务程序。

one loop per thread

这种模型下，程序的每个IO线程有一个event loop（或者叫Reactor），用于处理读写和定时时间。

优点

  • 线程数目基本固定，可以在程序启动的时候设置，不会频繁创建与销毁。
  • 可以很方便地在线程间调配负载。
  • IO事件发生的线程是固定的，同一个TCP连接不必考虑事件并发。

Event loop代表了线程的主循环，需要让哪个线程干活,就把timer或IO channel(如TCP连接)注册到哪个线程的loop里即可。对实时性有要求的connection可以单独用一个线程;数据量大的connection可以独占一个线程，并把数据处理任务分摊到另几个计算线程中(用线程池);其他次要的辅助性connections可以共享一个线程。

对于non-trivial（有一定复杂度的）的服务端程序，一般会采用non-blocking IO + IO multiplexing,每个connection/acceptor都会注册到某个event loop.上，程序里有多个event loop，每个线程至多有一个event loop。 多线程程序对event loop提出了更高的要求，那就是“线程安全”。要允许一个线程往别的线程的loop里塞东西7，这个loop必须得是线程安全的。

进程间通信只用TCP

为什么进程通信只用TCP？也就是Sockets，最大的好处在于：可以跨主机，具有伸缩性。

在编程上，TCP sockets和pipe都是操作文件描述符，用来收发字节流，都可以read/write/fcntl/select/poll等。不同的是，TCP是双向的，Linux的pipe是单向的，进程间双向通信还得开两个文件描述符，不方便;而且进程要有父子关系才能用pipe，这些都限制了pipe的使用。在收发字节流这一通信模型下，没有比Sockets/TCP更自然的IPC了。当然，pipe也有一个经典应用场景，那就是写Reactor/event loop时用来异步唤醒select ( 或等价的poll/epoll_wait) 调用。

分布式中使用TCP长连接通信

使用TCP长连接的好处有两点:

  • 容易定位分布式系统中的服务之间的依赖关系。只要在机器上运行netstat -tpna|grep:port就能立刻列出用到某服务的客户端地址( Foreign列),然后在客户端的机器上用netstat或lsof命令找出是哪个进程发起的连接。这样在迁移服务的时候能有效地防止出现outage。TCP短连接和UDP则不具备这一特性。
  • 通过接收和发送队列的长度也较容易定位网络或程序故障。在正常运行的时候，netstat 打印的Recv-Q和Send-Q都应该接近0，或者在0附近摆动。如果Recv-Q保持不变或持续增加，则通常意味着服务进程的处理速度变慢，可能发生了死锁或阻塞。如果Send-Q保持不变或持续增加，有可能是对方服务器太忙、来不及处理，也有可能是网络中间某个路由器或交换机故障造成丢包，甚至对方服务器掉线，这些因素都可能表现为数据发送不出去。
