Posix 消息队列

《Unix网络编程：卷2》``消息队列``posts

1 概述

消息队列可以认为是一个消息链表。有写权限的线程可往队列中放置消息，有读权限的线程可从队列中取走消息。

1.1 与System V的区别

应该写完System V再写这个。

  • 对Posix消息队列的读总是返回最高优先级的最早消息，对System V消息队列的读则可以返回指定优先级的消息；
  • 当往一个空队列放置一个消息时，Posix消息队列运行产生一个信号或启动一个线程，System V消息队列则不提供类似机制。

2 队列的基本操作

2.1 mq_open

#include <fcntl.h>           /* For O_* constants */
#include <sys/stat.h>        /* For mode constants */
#include <mqueue.h>

mqd_t mq_open(const char *name, int oflag);
mqd_t mq_open(const char *name, int oflag, mode_t mode, struct mq_attr *attr);

参数

  • name也是由自己定义的函数设定；
  • oflag参数和之前的一样，也是有O_RDONLY之类；
  • mode权限设置；
  • attr结构体设置属性，后面有。

返回值

mq_open的返回值称为消息队列描述符，和文件描述符或套接字描述符有点不一样。

注意！

使用Ubuntu系统运行书上的例子创建出来的queue不在/tmp目录下，而是在/dev/mqueue下。

在Linux上，posix 消息队列是以虚拟文件系统实现的，必须将其挂载到某个目录才能看见

2.2 mq_close

#include <mqueue.h>

int mq_close(mqd_t mqdes);

和close文件一样，关闭不代表消息队列已经从系统中删除了。

2.3 mq_unlink

#include <mqueue.h>

int mq_unlink(const char *name);

每个消息队列有一个保存其当前打开着描述符的引用计数器（和之前fork的时候引用计数+1是一个道理），当大于0的时候其name能删除。

Posix消息队列随内核，即使当前没有进程打开某个消息队列，该队列上的各个消息也一直存在。

3 attr属性

指向某个mq_attr结构的指针可做为mq_open的第四个参数传递，指定队列相关属性。

3.1 结构体

struct mq_attr {
      long mq_flags;       /* Flags: 0 or O_NONBLOCK */
      long mq_maxmsg;      /* Max. # of messages on queue */
      long mq_msgsize;     /* Max. message size (bytes) */
      long mq_curmsgs;     /* # of messages currently in queue */
};

3.2 mq_getattr函数

#include <mqueue.h>

int mq_getattr(mqd_t mqdes, struct mq_attr *attr);

获得消息队列默认属性。

3.3 mq_setattr函数

int mq_setattr(mqd_t mqdes, const struct mq_attr *newattr, struct mq_attr *oldattr);

设置消息队列的属性。

4 放置与取走

4.1 mq_send

#include <mqueue.h>
int mq_send(mqd_t mqdes, const char *msg_ptr, size_t msg_len, unsigned int msg_prio);

参数

前三个参数和write的类似，后面的prio为待发送消息的优先级，其值必须小于MQ_PRIO_MAX。

4.2 mq_receive

ssize_t mq_receive(mqd_t mqdes, char *msg_ptr, size_t msg_len, unsigned int *msg_prio);

可以看见返回值为ssize_t，取出的消息的大小。

5 消息队列限制

  • mq_mqxmsg 队列中的最大消息数；
  • mq_msgsize 给定消息的最大字节数；
  • MQ_OPEN_MAX 一个进程能够同时拥有的打开着消息队列的最大数目（Posix至少为8）；
  • MQ_PRIO_MAX 任意消息的最大优先级值加1（Posix至少为32）。

6 mq_notify函数

用来通知进程，消息队列里来消息了。这是System V不具备的功能。

#include <mqueue.h>
int mq_notify(mqd_t mqdes, const struct sigevent *sevp);

有两种通知方式，根据sigevent结构体中sigev_notify的值来决定：

  • 产生一个信号；
  • 创建一个线程来执行一个指定的函数。

结构体

struct sigevent{
    int sigv_notify;    //SIGEV_{NONE,SIGNAL,THREAD}
    int sigv_signo;
    union sigval sigv_value;
    .....
}

又称匿名管道，只能在同一程序里通信，比如父子进程，不能跨程序。

1.4 例子

小例子，2个子进程分别计算2-100的素数和101到200的素数，然后将结果通过管道发给主进程输出。

#include<stdio.h>
#include<unistd.h>
#include<stdlib.h>
#include<sys/stat.h>
#include<sys/wait.h>
#include<sys/types.h>
#include<fcntl.h>
#include<signal.h>

int numChild = 0;   //子进程数全局变量
void childProcEnd(int s)
{
    int status;

    if(s == SIGCHLD)
    {
        printf("delete child%d\n",numChild);
        numChild++;
        wait(&status);
    }       
    if(numChild == 2)
    {
        printf("all child delete\n");
        exit(0);
    }
}
int isSushu(int num)
{
    int tp = num;
    for(int i = 2;i < tp;i++)
    {
        if(num%i == 0) return 0;
    }
    return 1;
}

int main()
{   
    int numProc = 2;
    int fds[2];
    pipe(fds);
    int a,b;
    signal(SIGCHLD,childProcEnd);
    printf("father id:%d\n",getpid());
    while(1)
    {
        if(numProc == 2)
        {
            a = 2;
            b = 100;
        }
        if(numProc == 1)
        {
            a = 101;
            b = 200;
        }
        if(fork() > 0)//在主进程里创建
        {
            if(--numProc == 0)
                break;
        }
        else
        {
            close(fds[0]);
            for(int i = a;i <= b;i++)
            {
                if(isSushu(i) == 1)
                {
                    write(fds[1] ,&i, 4);
                    printf("%d\n",i);
                }
            }
            exit(0);//当子进程事情做完后退出,是退出的子进程而不是父进程
            //子进程退出，等待主进程的回收，以防僵死进程
            //break;
        }
    }

    {
        close(fds[1]); //把写关闭
        int wfd = open("result.txt",O_RDWR | O_CREAT);
        while(1)
        {
            int num = 0;
          int ret = read(fds[0],&num, 4);
            if(ret == 4)
                write(wfd,&num,4);
            printf("father %d\n",num);
        }
    }
    return 0;   
}

2 FIFO

之前的管道没有名字，因此它们的最大劣势是只能用于有一个共同祖先进程的各个进程之间。

FIFO指代先进先出（first in，first out），FIFO类似与管道。它是一个单向（半双工）数据流。不同于管道的是，每个FIFO有一个路径名与之关联，从而运行无亲缘关系的进程访问同一个FIFO。

FIFO也称有名管道。

2.1 mkfifo函数

#include <sys/types.h>
#include <sys/stat.h>
int mkfifo(const char *pathname, mode_t mode);

pathname为文件名，一般是新建。mode为权限位，比如0666权限为可读可写权限，且三个用户组都可以读写。

mkfifo("2.pipe",0666); //0666可读写权限

mkfifo也可以在shell里使用。

创建出一个FIFO后，它只能读或者写，不能同时进行。可以用open打开也可以用标准I/O函数，如fopen打开。

对管道或FIFO的write总是往末尾添加数据，对它们的read则总是从开头返回数据。

注意：如果当前尚没有任何进程写打开某个FIFO，那么读打开该FIFO的进程将阻塞。同理，如果当前尚没有任何进程读打开某个FIFO，那么写打开该FIFO的进程将阻塞。

3 非阻塞

和套接字一样，管道也可以设置非阻塞。

3.1 设置方式

（1）open指定

调用open时可指定O_NONBLOCK标志。例如：

wtirefd = open(FIFO!, O_WRONLY | O_NONBLOCK, 0);

（2）fcntl设置

如果描述符已经打开，那么可以调用fcntl以启动O_NONBLOCK标志。对于匿名管道来说，必须使用这种技术，因为匿名管道没有open调用，在pipe调用中也无法指定O_NONBLOCK。设置方法和socket一样：

int flags;

if (( flags = fcntl(fd, F_GETFL, 0)) < 0)
    err_sys();
flags |= O_NONBLOCK;
fcntl(fd, F_SETFL, flags);

1 概述

多线程一直是计算机中一个重要的概念，有关于线程的基础概念就不再赘述。 之前写的多任务处理都是用的fork，但是fork的调用太浪费资源，而且父子进程之前通信需要花更多功夫。 线程有助于解决以上问题。 本章主要讲解一下在Unix中实现多线程、互斥锁和条件变量。

2 线程的创建

程序运行就会有一个主线程，其余线程由pthread_create函数创建。

2.1 函数原型

#include <pthread.h>
int pthread_create(pthread_t thread, const pthread_attr_t *attr,
void *(start_routine) (void *), void *arg);

2.2 线程的属性

每个线程都有许多属性：

  • 优先级
  • 初始栈大小
  • 是否应该成为一个守护进程

可以通过pthread_attr_t变量指定这些属性，一般都置为空。

2.3 函数传入线程

Unix中的线程和之前写过的QT&&C++中的线程不一样，前者是将函数放入线程中运行，后者是创建一个线程，在run函数里的内容就是在线程里跑的内容。

创建一个线程时我们最后指定的参数是由该线程执行的函数以及参数。该线程通过调用这个函数开始执行，然后显式的终止（调用pthread_exit），或者隐式地终止（函数返回）。 要注意函数的唯一调用参数是指针*arg，如果需要传递多个参数则得打包成一个结构体。也可以将参数定义为全局变量，以便于函数使用。

3 线程主要函数

3.1 pthread_join

#include <pthread.h>
int pthread_join(pthread_t thread, void **retval);

类似于进程里的waitpid，等待给定线程终止。如果该线程函数没有结束就调用了pthread_join，调用者将会被阻塞。

3.2 pthread_self

#include <pthread.h>
pthread_t pthread_self(void);

类似于getpid，获取自身线程ID。

3.3 pthread_detach

include <pthread.h>
int pthread_detach(pthread_t thread);

把指定线程转变为脱离状态。 结合join函数的问题来看，如果不希望调用者堵塞，可以在子线程中如下加入代码：

pthread_detach(pthread_self());
//或者父线程调用
pthread_detach(thread_id);
//非阻塞，可立即返回

这将该子线程的状态设置为detached,则该线程运行结束后会自动释放所有资源。

3.4 pthread_exit

#include <pthread.h>
void pthread_exit(void *retval);

终止线程的方法之一。 如果本线程未曾脱离（detach），它的线程ID和退出状态将一直留存到调用进程内的某个其他线程对它调用pthread_join。

具体实现例子可以看书。

4 互斥锁

线程编程是并发编程或并行编程，多个线程可以并发的（并行的）运行且访问相同的变量。但是在某些情况，线程编程访问变量会出错。而进程则不会，因为父子进程之前除了描述符外不共享任何东西。 这个时候就需要使用互斥锁解决这个问题，可能有学过操作系统里银行家算法的想必对互斥锁不陌生。

4.1 相关函数

int pthread_mutex_lock(pthread_mutex_t *mutex);

int pthread_mutex_trylock(pthread_mutex_t *mutex);

int pthread_mutex_unlock(pthread_mutex_t *mutex);

int pthread_mutex_destroy(pthread_mutex_t *mutex);

*注意：如果对一个已经被另外线程锁住的互斥锁上锁，那么本线程将被阻塞，直到该互斥锁被解锁位置。

4.2 初始化

pthread_mutex_t counter_mutex = PTHREAD_MUTEX_INITIALIZER;

有些系统将PTHREAD_MUTEX_INITIALIZER定义为0。

4.3 上锁解锁过程

书上的例子：

for (i = 0; i < NLOOP; i++) {
        Pthread_mutex_lock(&counter_mutex);

        val = counter;
        printf("%d: %d\n", pthread_self(), val + 1);
        counter = val + 1;

        Pthread_mutex_unlock(&counter_mutex);
    }

可见，lock和unlock之间的变量是互斥的，别的线程不能同时访问。

5 条件变量

互斥锁适合与防止同时访问某个共享变量，但是我们需要另外某种在等待某个条件发生期间能让我们进入睡眠的东西。简单来说就是让主循环进入睡眠，知道某个线程通知它有事可做才醒来的方法。 条件变量结合互斥锁能够提供这个功能。互斥锁提供互斥机制，条件变量提供信号机制。

5.1 相关函数

#include <pthread.h>
//条件变量
pthread_cond_t cond = PTHREAD_COND_INITIALIZER;

int pthread_cond_init(pthread_cond_t *cond, pthread_condattr_t *cond_attr);

int pthread_cond_signal(pthread_cond_t *cond);

int pthread_cond_broadcast(pthread_cond_t *cond);

int pthread_cond_wait(pthread_cond_t *cond, pthread_mutex_t *mutex);

int pthread_cond_timedwait(pthread_cond_t *cond,  pthread_mutex_t  *mutex,  const struct timespec *abstime);
int pthread_cond_destroy(pthread_cond_t *cond);

5.2 函数解释

初始化和销毁就不说了。

以下内容来自Linux线程同步之条件变量

1）pthread_cond_signal() & pthread_cond_broadcast()

这两个函数都是用于向等待条件的线程发送唤醒信号，pthread_cond_signal()函数只会唤醒等待该条件的某个线程，pthread_cond_broadcast()会广播条件状态的改变，以唤醒等待该条件的所有线程。例如多个线程只读共享资源，这是可以将它们都唤醒。

2）pthread_cond_wait() & pthread_cond_timedwait()

函数用于等待条件被触发。该函数传入两个参数，一个条件变量一个互斥量，函数将条件变量和互斥量进行关联，互斥量对该条件进行保护，传入的互斥量必须是已经锁住的。调用pthread_cond_wait()函数后，会原子的执行以下两个动作：

  • 将调用线程放到等待条件的线程列表上，即进入睡眠；
  • 对互斥量进行解锁；

pthread_cond_timedwait()函数和和pthread_cond_wait()的工作方式相似，只是多了一个等待时间。等待时间的结构为struct timespec。

struct timespec{  
time_t  tv_sec    //Seconds.  
long    tv_nsec   //Nanoseconds.  
};

}sharedData = {PTHREAD_MUTEX_INITIALIZER, PTHREAD_COND_INITIALIZER};  

void * produce(void *ptr)  
{  
    for (int i = 0; i < 10; ++i)  
    {  
        pthread_mutex_lock(&sharedData.mutex);  //生产的时候将product锁住

        sharedData.product.push(i);  

        pthread_mutex_unlock(&sharedData.mutex);  

        if (sharedData.product.size() == 1)     //生产了一个产品则唤醒消费线程
            pthread_cond_signal(&sharedData.cond);  

        //sleep(1);  
    }  
}  

void * consume(void *ptr)  
{  
    for (int i = 0; i < 10;)  
    {  
        pthread_mutex_lock(&sharedData.mutex);  

        while(sharedData.product.empty())  //消费完了将进程放入睡眠状态
            pthread_cond_wait(&sharedData.cond, &sharedData.mutex);  

        ++i;  
        cout<<"consume:"<<sharedData.product.front()<<endl;  
        sharedData.product.pop();  

        pthread_mutex_unlock(&sharedData.mutex);  

        //sleep(1);  
    }  
}  

int main()  
{  
    pthread_t tid1, tid2;  

    pthread_create(&tid1, NULL, consume, NULL);  
    pthread_create(&tid2, NULL, produce, NULL);  

    void *retVal;  

    pthread_join(tid1, &retVal);  
    pthread_join(tid2, &retVal);  

    return 0;  
}

6 条件变量与互斥锁、信号量的区别

  1. 互斥锁必须总是由给它上锁的线程解锁，信号量的挂出即不必由执行过它的等待操作的同一进程执行。一个线程可以等待某个给定信号灯，而另一个线程可以挂出该信号灯。

  2. 互斥锁要么锁住，要么被解开（二值状态，类型二值信号量）。

  3. 由于信号量有一个与之关联的状态（它的计数值），信号量挂出操作总是被记住。然而当向一个条件变量发送信号时，如果没有线程等待在该条件变量上，那么该信号将丢失。

  4. 互斥锁是为了上锁而设计的，条件变量是为了等待而设计的，信号灯即可用于上锁，也可用于等待，因而可能导致更多的开销和更高的复杂性。

非阻塞式I/O

《Unix网络编程：卷1》``posts

将I/O设置为非阻塞是提高传输效率的重要手段之一。

1 概述

一开始的笔记有写到将套接字设置成非阻塞式，但是好像只是设置了，并没有应用。这一章主要讲一下如何实现非阻塞式I/O以提高效率。 可能阻塞的套接字调用可分以下四类：

  • 输入操作，包括read、readv、recv、recvfrom、和recvmsg；
  • 输出操作，包括write、writev、send、sendto和sendmsg；
  • 接收外来连接，即accept函数；
  • 发起外出连接，即connect函数。

2 非阻塞读写

书上第一个例子太冗长了，简单理解就是在读和写的两个缓冲区设置了几个指针，指示了当前写容量和可读容量，并对其进行操作。 主要是应用了select实现的非阻塞I/O。 简单一点的是使用多进程，一个进程专门用来读，一个进程用来写。但是要注意异步，用信号通知另一进程做相应的操作。

3 非阻塞connect

当在一个非阻塞的TCP套接字上调用connect时，connect将立即返回一个EINPROGRESS错误，不过已经发起的TCP三路握手继续进行。然后使用select检测是否成功。 具体代码看书353页。

4 非阻塞accept

4.1 阻塞的accept

之前有个猜想，为什么服务器里的while循环没有一直运行，而是有连接才进行下一步操作。后来发现accept函数是阻塞的。

来自man 2 accept里的资料：

If no pending connections are present on the queue, and the socket is not marked as nonblocking,  accept() blocks  the  caller until a connection is present.  If the socket is marked nonblocking and no pending connections are present on the queue, accept() fails with the error EAGAIN or EWOULDBLOCK.

可见如果没有将套接字设置为非阻塞，那么accept会一直等待下一个连接，从而accept下面的语句就不会执行。

4.2 非阻塞

和很多非阻塞函数一样，要处理返回的错误信息，甚至可以忽略这些错误信息。 主要是使用select加非阻塞套接字实现非阻塞。 具体看书362页。

套接字和标准I/O

《Unix网络编程：卷1》``SOCKET``posts 之前一直使用的Unix I/O，包括read，write这两个函数以及它们的变体（recv、send等）。下面使用标准I/O库（ANSI C）实现读取写入。

1 fdopen函数

#include <stdio.h>
FILE *fdopen(int fd, const char *mode);

返回值是文件流形式。 .0

1.1 模式

fdopen有模式区分，比如：

  • r，读
  • r+，读和写
  • w，写
  • w+，写和读
  • a，append，在文件末端写
  • a+，可读，在文件末端写

1.2 例子

char        line[MAXLINE];
    FILE        *fpin, *fpout;

    fpin = Fdopen(sockfd, "r");
    fpout = Fdopen(sockfd, "w");

    while (Fgets(line, MAXLINE, fpin) != NULL)
        Fputs(line, fpout);

该例子是完全缓冲的，不能实时回显。

1.3 三类缓冲

标准I/O函数库执行以下三类缓冲：

1）完全缓冲

只在以下情况发生I/O：

  • 缓冲区满，通常为8192字节；
  • 进程显式调用fflush（之前有写过的将缓冲区写入的函数，还有类似的fseek）；
  • 进程调用exit。

2）行缓冲

只在以下情况发生I/O：

  • 碰到一个换行符；
  • 进程调用fflush；
  • 进程调用exit。

3）不缓冲

每次调用标准I/O输出函数都发生I/O。 标准I/O函数库的大多数Unix实现使用如下规则：

  • 标准错误输出（2）总是不缓冲；
  • 标准输入（0）和标准输出（1）完全缓冲，除非它们指代终端设备（行缓冲）；
  • 所有其他I/O流都是完全缓冲，除非它们指代中断设备（行缓冲）。

2 弊端

如上述问题所知，其实套接字上是不建议使用标准I/O。但是也有办法解决，比如：

  • 通过调用setvbuf迫使这个输出流变为行缓冲；
  • 在每次调用fputs之后通过调用fflush强制输出每个缓冲区的字符。

基本UDP套接字编程

《Unix网络编程：卷1》``udp``posts

udp的特性就不再介绍。

1 概述

主要流程：

2 recvfrom和sendto函数

UDP不同于TCP，客户不需要与服务器建立连接，只需要指定目的地址就可以发送数据。发送数据使用sendto，接收使用recvfrom。这两个函数类似于标注的read和write。

2.1 函数原型

#include <sys/socket.h>
ssize_t recvfrom(int sockfd, void *buf, size_t len, int flags,
                        struct sockaddr *src_addr, socklen_t *addrlen);
ssize_t sendto(int sockfd, const void *buf, size_t len, int flags,
                      const struct sockaddr *dest_addr, socklen_t addrlen);

2.2 参数详解

前三个参数就不再介绍了，等同与read和write函数的三个参数。

1）flags

一般置为0.

2）src_addr和 addrlen

  • recvfrom的最后两个参数类似与accept的最后两个参数：返回其中套接字地址结构的内容告诉我们是谁发送了数据报（UDP）或是谁发起了连接（TCP）。
  • sendto的最后两个参数类似与connect的最后两个参数：调用时其中套接字地址结构被我们跳入数据报将发往（UDP）或与之建立连接（TCP）的协议地址。

3 例子

服务端：

#include    "unp.h"

int
main(int argc, char **argv)
{
    int                 sockfd;
    struct sockaddr_in  servaddr, cliaddr;

    sockfd = Socket(AF_INET, SOCK_DGRAM, 0); //注意这里是SOCK_DGRAM,而TCP是SOCK_STREAM。

    bzero(&servaddr, sizeof(servaddr));
    servaddr.sin_family      = AF_INET;
    servaddr.sin_addr.s_addr = htonl(INADDR_ANY);
    servaddr.sin_port        = htons(SERV_PORT);

    Bind(sockfd, (SA *) &servaddr, sizeof(servaddr));

    dg_echo(sockfd, (SA *) &cliaddr, sizeof(cliaddr));//函数实现可看书
}

客户端：

#include    "unp.h"

int
main(int argc, char **argv)
{
    int                 sockfd;
    struct sockaddr_in  servaddr;

    if (argc != 2)
        err_quit("usage: udpcli <IPaddress>");

    bzero(&servaddr, sizeof(servaddr));
    servaddr.sin_family = AF_INET;
    servaddr.sin_port = htons(SERV_PORT);
    Inet_pton(AF_INET, argv[1], &servaddr.sin_addr);

    sockfd = Socket(AF_INET, SOCK_DGRAM, 0);

    dg_cli(stdin, sockfd, (SA *) &servaddr, sizeof(servaddr));

}

套接字选项

《Unix网络编程：卷1》``posts``SOCKET

这一章主要介绍一下几个函数。

1 getsockopt函数和setsockopt函数

  • getsockopt()函数用于获取任意类型、任意状态套接口的选项当前值，并把结果存入optval。在不同协议层上存在选项，但往往是在最高的“套接口”层次上，设置选项影响套接口的操作，诸如操作的阻塞与否、包的选径方式、带外数据的传送等。

  • setsockopt()函数用于任意类型、任意状态套接口的设置选项值。尽管在不同协议层上存在选项，但本函数仅定义了最高的“套接口”层次上的选项。选项影响套接口的操作，诸如加急数据是否在普通数据流中接收，广播数据是否可以从套接口发送等等。

1.1 函数原型

#include <sys/types.h>          /* See NOTES */
#include <sys/socket.h>

int getsockopt(int sockfd, int level, int optname,void *optval, socklen_t *optlen);
int setsockopt(int sockfd, int level, int optname, const void *optval, socklen_t optlen);

1.2 参数详解

  • sockfd：一个标识套接口的描述字。
  • level：选项定义的层次。支持的层次仅有SOL_SOCKET和IPPROTO_TCP。
  • optname：需获取的套接口选项。
  • optval：指针，对于getsockopt()，指向返回选项值的缓冲。对于setsockopt()，指向包含新选项值的缓冲。
  • optlen：指针，对于getsockopt()，作为入口参数时，选项值的最大长度。作为出口参数时，选项值的实际长度。对于setsockopt()，现选项的长度。

2 fcntl函数

可执行各种描述符控制操作，比如设置非阻塞式I/O。

2.1 函数原型

#include <unistd.h>
#include <fcntl.h>
int fcntl(int fd, int cmd); 
int fcntl(int fd, int cmd, long arg); 
int fcntl(int fd, int cmd, struct flock *lock);

返回：若成功则取决于cmd，若出错则为-1。 下列三个命令有特定返回值：

  • F_DUPFD 返回新的文件描述符
  • F_GETFD 返回相应标志
  • F_GETFL , F_GETOWN 返回一个正的进程ID或负的进程组ID

2.2 参数详解

主要看一下cmd的值：

  • F_SETFL 设置文件状态标记
  • F_GETFL 获得文件状态标记
  • F_DUPFD 复制一个现有的描述符
  • F_GETFD 获得文件描述符标记
  • F_SETFD 设置文件描述符标记
  • F_GETOWN 获得异步I/O所有权
  • F_SETOWN 设置I/O所有权

主要影响套接字描述符的两个标志是：

  • O_NONBLOCK—非阻塞式I/O；
  • O_ASYNC—信号驱动式I/O；

2.3 设置非阻塞

int flags

if((flags = fcntl(fd, F_GETFL, 0)) < 0)
    err_sys("F_GETFL error");
flags |= O_NONBLOCK;
if (fcntl(fd, F_SETFL, flags) < 0)
    err_sys("F_SETFL error");

不能图方便而直接使用以下代码：

if(fcntl(fd, F_SETFL, O_NONBLOCK) < 0)
    err_sys("F_SETFL error");

这段代码在设置非阻塞标志的同时也清楚了所有其他文件状态标志。

3 几个套接字有用的选项

3.1 SO_REUSEADDR

四个不同的功能：

  • 允许启动一个监听服务器并捆绑其众所周知的端口；
  • 允许在同一端口上启动同一服务器的多个实例；
  • 允许单个进程捆绑同一端口到多个套接字上，只要每次捆绑指定不同的本地IP地址即可；
  • 运行完全重复的捆绑：当一个IP地址和端口已绑定到某个套接字上时，只有UDP支持将同样的IP地址和端口捆绑到另一个套接字上。

使用实例：

Setsockopt(listenfd, SOL_SOCKET, SO_REUSEADDR, &on, sizeof(on));

3.2 SO_RCVBUF和SO_SNDBUF

更改套接字接收缓冲区和发送缓冲区的大小。 使用实例：

int n = 220 * 1024;
Setsockopt(listenfd, SOL_SOCKET, SO_RCVBUF, &on, sizeof(on));

select函数

《Unix网络编程：卷1》``select``posts

第六章主要讲的是I/O复用的2个函数，select和poll函数。现在主要看看select函数

1 函数原型

#include <sys/select.h>
/* According to earlier standards */
#include <sys/time.h>
#include <sys/types.h>
#include <unistd.h>

int select(int nfds, fd_set *readfds, fd_set *writefds,
          fd_set *exceptfds, struct timeval *timeout);

该函数运行进程指示内核等待多个事件中的任何一个发生，并只在有一个或多个时间发生或经历一段指定的时间后才唤醒它。 简单说，使用select就可以实现非阻塞，一旦执行必然会有返回值，以返回值的不同来反映函数的执行情况。

2 参数解析

这几个参数其实都不是很重要

2.1 nfds

是一个整数值，是指集合中所有文件描述符的范围，即所有文件描述符的最大值加1。每次循环，描述符0，1，2，···，一直到nfds-1都会被测试。

怪不得有一个maxfdp1 = max(fileno(fp), sockfd) + 1的语句

2.2 *readfds

指向fd_set结构的指针，监视文件描述符的读变化。如果集合中有一个文件可读，select就会返回一个大于0的值，表示有文件可读。如果没有那么根据timeout参数判断是否超时。超出返回0，错误返回负值。可以传入NULL值，表示不关心任何文件的读变化。

2.3 *writefds

上同，改为写变化。

2.4 *exceptfds

用来观察异常情况。

上述三个参数都值—结果参数。调用select函数时，我们指定所关心的描述符的值，函数返回时，结果将指示哪些描述符已就绪。

2.5 *timeout

告知内核等待所指定描述符中的任何一个就绪可花多长时间。

3 宏定义

有几个宏定义需要了解一下。

3.1 FD_ZERO(fd_set *rset)

初始化文件描述符集合。

3.2 FD_SET(int fd, fd_set *fdset)

将描述符放入集合。

3.3 FD_CLR(int fd, fd_set *fdset)

将描述符移除集合。

3.4 FD_ISSET(int fd, fd_set *fdset)

判断描述符是否在集合中。

4 描述符就绪条件

4.1 套接字准备读

满足以下任一情况则可读：

  • 该套接字接受缓冲区中的数据字节数大于等于套接字接收缓冲区低水位标记的当前大小；
  • 该连接的读半关闭（接收了FIN的TCP连接）；
  • 该套接字是一个监听套接字且已完成的连接数不为0；
  • 其上有一个套接字错误待处理。

4.2 套接字准备写

满足以下任一情况则可写：

  • 该套接字发送缓冲区中的可用空间字节数大于等于套接字发送缓冲区低水位标记的当前大小，并且或者该套接字已连接，或者该套接字不需要连接（如UDP套接字）；
  • 该连接的写半部关闭，对这样的套接字进行写操作会产生SIGPIPE信号；
  • 使用非阻塞式connect的套接字已建立连接；
  • 其上只有一个套接字错误待处理。
